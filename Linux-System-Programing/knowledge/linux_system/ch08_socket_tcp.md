# TCP/IP Fundamentals & Internet Domain Sockets

## Problem It Solves

Topic 8.1 covered the socket API mechanics. But to write correct Internet domain socket code,
we need to understand **why** `htons()` must be called, what `sockaddr_in` fields mean, and
why TCP is "reliable" while UDP is not. Without this foundation, bugs like silent byte-order
corruption or incorrect address initialization are easy to introduce and hard to diagnose.

This topic covers two complementary areas:
- **TCP/IP Fundamentals** (Ch58): the protocol stack, IP addressing, port numbers, UDP vs TCP
- **Internet Domain Sockets** (Ch59): address structures, byte-order conversion, `getaddrinfo()`

## Concept Overview

### TCP/IP Protocol Stack — Layered Model

```
┌──────────────────────────────────┐
│  Application layer               │  Your code: HTTP, DNS, Redis protocol...
├──────────────────────────────────┤
│  Transport layer                 │  TCP (SOCK_STREAM) / UDP (SOCK_DGRAM)
├──────────────────────────────────┤
│  Network layer                   │  IP — routing, addressing, fragmentation
├──────────────────────────────────┤
│  Data-link layer                 │  Ethernet driver + NIC hardware
└──────────────────────────────────┘
```

**Encapsulation principle**: each layer wraps the layer above's data as opaque bytes, adds
its own header, and passes the result down. On receive, each layer strips its header and
passes data up:

```
App data
  → [TCP header | App data]             = TCP segment
  → [IP  header | TCP segment]          = IP datagram
  → [Eth header | IP datagram | CRC]    = Ethernet frame
```

### IP — Connectionless, Unreliable, Best-effort

IP handles **routing** (getting packets to the right machine) but provides **no guarantees**:
- Packets may arrive **out of order**
- Packets may be **lost** (router buffer overflow)
- Packets may be **duplicated** (retransmit + reorder at data-link layer)
- No error recovery (corrupted headers → silently discarded)

**Fragmentation**: Ethernet MTU = 1500 bytes. IPv4 datagrams can be up to 65535 bytes. When
a datagram exceeds the MTU of a link, IP **fragments** it automatically. Fragmentation is
undesirable — losing one fragment loses the entire datagram. TCP uses **Path MTU Discovery**
to avoid it. UDP applications must self-limit size (practical limit: ≤ 512 bytes payload).

### Port Numbers — Transport-layer Multiplexing

IP address identifies **which machine**. Port number identifies **which process** on that machine.

```
Port range        Category
──────────────────────────────────────────────────────────
0–1023            Well-known (CAP_NET_BIND_SERVICE required)
                  SSH=22, SMTP=25, DNS=53, HTTP=80, HTTPS=443
1024–41951        IANA registered ports
49152–65535       Dynamic/ephemeral (IANA designation)

Linux ephemeral range: /proc/sys/net/ipv4/ip_local_port_range
                       default: 32768–60999
```

A TCP connection is uniquely identified by the **4-tuple**:
```
(src_ip, src_port, dst_ip, dst_port)
```
This is why `accept()` creates a new socket for each connection — each has a unique 4-tuple.

Multiple connections to the same server port are possible because the 4-tuple differs in
`src_ip:src_port` for each client.

### UDP — Port Numbers + Checksum on Top of IP

UDP adds only two features to IP:
1. **16-bit port numbers** (source + destination)
2. **16-bit checksum** (optional in IPv4, mandatory in IPv6)

No connection, no ACK, no ordering, no flow control. Fast and simple. Used when:
- Latency matters more than reliability (DNS, video streaming, gaming)
- Application implements its own reliability (QUIC over UDP)
- Broadcast / multicast is needed

**UDP datagram size**: to avoid IP fragmentation, limit payload to ≤ 548 bytes
(576 - 20 IP header - 8 UDP header). Many applications use ≤ 512 bytes.

### TCP — Reliable Stream Over Unreliable IP

TCP adds the following mechanisms on top of IP to provide a reliable byte-stream:

| Mechanism | Solves |
|-----------|--------|
| **3-way handshake** | Establishes connection, synchronizes ISN both ways |
| **Sequencing** | Each byte gets a sequence number → correct reassembly |
| **ACK + retransmission** | Sender retransmits if ACK not received before timeout |
| **Flow control** (sliding window) | Receiver advertises buffer size → sender won't overflow it |
| **Congestion control** (slow-start + congestion avoidance) | Sender probes network capacity, avoids overloading routers |
| **4-way FIN exchange** | Graceful bidirectional shutdown |

**Why ISN ≠ 0**: Initial Sequence Number is generated by an algorithm that increments over
time and adds randomness. This prevents "old duplicate" segments from a previous incarnation
of the same 4-tuple being mistaken for valid current data, and makes ISN prediction
attacks harder.

**Nagle Algorithm**: TCP buffers small `write()` calls and coalesces them into one segment
to reduce packet count. Beneficial for bulk transfers, but adds latency for interactive
protocols. Disable with `TCP_NODELAY` when low latency matters:
```c
int flag = 1;
setsockopt(sockfd, IPPROTO_TCP, TCP_NODELAY, &flag, sizeof(flag));
```

### Byte Order — Critical When Writing sockaddr Structures

Different CPU architectures store multi-byte integers differently:
- **Little-endian** (x86, x86-64): least significant byte at lowest address
- **Big-endian**: most significant byte at lowest address (= **network byte order**)

Port 8080 = `0x1F90`:
```
                   byte @addr N    byte @addr N+1
little-endian:        0x90            0x1F        ← x86 RAM layout
big-endian (network): 0x1F            0x90        ← what the network expects
```

If a little-endian machine copies port `8080` into `sin_port` without conversion, a
big-endian receiver interprets it as `0x901F = 36895`. **Silent data corruption.**

**Four mandatory conversion functions:**

```c
#include <arpa/inet.h>

uint16_t htons(uint16_t v);   /* host → network, 16-bit  (port numbers) */
uint32_t htonl(uint32_t v);   /* host → network, 32-bit  (IPv4 address constants) */
uint16_t ntohs(uint16_t v);   /* network → host, 16-bit */
uint32_t ntohl(uint32_t v);   /* network → host, 32-bit */
```

**Name mnemonic**: `h` = host, `n` = network, `s` = short (16-bit), `l` = long (32-bit).

On big-endian machines, these are no-ops. Always call them anyway for portability.

## System Context

### Where These Structures Live in the Stack

```
Application code
  │  struct sockaddr_in / sockaddr_in6
  │
  ▼
socket() / bind() / connect() / accept()
  │  kernel validates and stores address
  ▼
TCP/IP stack (kernel)
  │  TCP header: src_port, dst_port, seq, ack, flags, window
  │  IP  header: src_addr, dst_addr, TTL, protocol, checksum
  ▼
NIC driver → network wire
```

### DNS and Name Resolution

Humans use hostnames (`example.com`), machines use binary IP addresses. The chain:

```
Application                  DNS resolver               DNS servers
  getaddrinfo("example.com")
       │
       ▼
  check /etc/hosts
       │  (if not found)
       ▼
  query /etc/resolv.conf nameserver
       │
       ▼
                             recursive DNS query → authoritative NS
                                                      → returns A/AAAA records
       ◄──────── binary IP addresses (may be multiple) ────────────────────
  returns linked list of addrinfo structs
```

`getaddrinfo()` handles this entire chain transparently — including `/etc/hosts`,
`/etc/nsswitch.conf`, DNS, and mDNS depending on system configuration.

## Architecture

### Internet Address Structures

**IPv4 — `struct sockaddr_in`:**

```c
#include <netinet/in.h>

struct in_addr {
    in_addr_t s_addr;        /* 32-bit IPv4 address, NETWORK byte order */
};

struct sockaddr_in {
    sa_family_t    sin_family;  /* AF_INET */
    in_port_t      sin_port;    /* 16-bit port, NETWORK byte order */
    struct in_addr sin_addr;    /* 32-bit IPv4 addr, NETWORK byte order */
    unsigned char  __pad[8];    /* padding to 16 bytes (= sizeof sockaddr) */
};
```

**IPv6 — `struct sockaddr_in6`:**

```c
struct in6_addr {
    uint8_t s6_addr[16];        /* 128-bit IPv6 address */
};

struct sockaddr_in6 {
    sa_family_t     sin6_family;   /* AF_INET6 */
    in_port_t       sin6_port;     /* 16-bit port, NETWORK byte order */
    uint32_t        sin6_flowinfo; /* flow info — set to 0 */
    struct in6_addr sin6_addr;     /* 128-bit IPv6 address */
    uint32_t        sin6_scope_id; /* scope ID — set to 0 */
};
```

**Generic — `struct sockaddr_storage`:**

```c
struct sockaddr_storage {
    sa_family_t ss_family;
    char        __ss_padding[118]; /* large enough for any address family */
};
/* Use when code must handle both IPv4 and IPv6 without knowing which */
```

### Special IPv4 Addresses

| Constant | Value | Meaning |
|----------|-------|---------|
| `INADDR_ANY` | `0.0.0.0` | Bind to all interfaces on this host |
| `INADDR_LOOPBACK` | `127.0.0.1` | Loopback — stays within kernel |
| `INADDR_BROADCAST` | `255.255.255.255` | Broadcast to local subnet |

IPv6 equivalents: `in6addr_any` (`::`) and `in6addr_loopback` (`::1`).

### Address Conversion API

**Obsolete** (appear in legacy code, do not use in new code):
`inet_aton()`, `inet_ntoa()`, `gethostbyname()`, `getservbyname()`

**Modern:**

```c
#include <arpa/inet.h>

/* String "204.152.1.2" or "::1" → binary in network byte order */
int inet_pton(int af, const char *src, void *dst);
/* Returns: 1=success, 0=not valid presentation, -1=error */

/* Binary → string (thread-safe, handles IPv4 and IPv6) */
const char *inet_ntop(int af, const void *src, char *dst, socklen_t len);
/* len = INET_ADDRSTRLEN (16) for IPv4, INET6_ADDRSTRLEN (46) for IPv6 */
```

### `getaddrinfo()` — Modern DNS + Port Lookup

```c
#include <netdb.h>

int getaddrinfo(const char *host,          /* hostname or IP string; NULL + AI_PASSIVE = wildcard */
                const char *service,       /* port number or name ("http", "80") */
                const struct addrinfo *hints,
                struct addrinfo **result); /* linked list of results */

void freeaddrinfo(struct addrinfo *result);
const char *gai_strerror(int errcode);     /* getaddrinfo has its own error codes */

struct addrinfo {
    int              ai_flags;      /* AI_PASSIVE, AI_NUMERICHOST, ... */
    int              ai_family;     /* AF_INET, AF_INET6, AF_UNSPEC */
    int              ai_socktype;   /* SOCK_STREAM, SOCK_DGRAM */
    int              ai_protocol;
    socklen_t        ai_addrlen;
    struct sockaddr *ai_addr;       /* pointer to sockaddr_in or sockaddr_in6 */
    char            *ai_canonname;
    struct addrinfo *ai_next;       /* next address in linked list */
};
```

**Why a linked list?** A hostname can resolve to multiple addresses:
- Dual-stack: A record (IPv4) + AAAA record (IPv6)
- Multi-homed server: multiple NICs
- DNS round-robin: same name → multiple IPs for load balancing

Always iterate the list and try each entry until `connect()` (or `bind()`) succeeds.

## Execution Flow

### TCP Connection via `getaddrinfo()` (Client)

```
getaddrinfo("example.com", "80", &hints, &res)
  │
  ▼ returns linked list:
  [AF_INET6, ::1:80] → [AF_INET, 93.184.216.34:80] → NULL

for rp = res; rp != NULL; rp = rp->ai_next:
  sockfd = socket(rp->ai_family, rp->ai_socktype, 0)
  connect(sockfd, rp->ai_addr, rp->ai_addrlen)
    │  success → break
    │  fail    → close(sockfd), try next
    ▼
freeaddrinfo(res)     ← always free, even on failure

read() / write()
close(sockfd)
```

### Byte Order Flow When Filling `sockaddr_in`

```
Application (host byte order)         sockaddr_in (network byte order)
─────────────────────────────────────────────────────────────────────
port = 8080                →  sin_port  = htons(8080)   = 0x1F90
"192.168.1.1"              →  sin_addr  = inet_pton(...)
INADDR_ANY (= 0)           →  sin_addr  = htonl(INADDR_ANY)

Rule: everything written into sockaddr fields MUST be in network byte order.
      Everything read from sockaddr fields MUST be converted with ntohs/ntohl.
```

### TCP Congestion Control — Slow Start

```
Connection opened:
  cwnd = 10 × MSS (initial window, Linux default)
  ssthresh = 65535 (initial)

Each ACK received (slow start phase):
  cwnd += 1 MSS  →  cwnd doubles per RTT (exponential growth)

When cwnd >= ssthresh (congestion avoidance):
  cwnd += MSS²/cwnd per ACK  →  +1 MSS per RTT (linear growth)

On packet loss (timeout):
  ssthresh = cwnd / 2
  cwnd = 1 MSS   → restart slow start

On 3 duplicate ACKs (fast retransmit):
  ssthresh = cwnd / 2
  cwnd = ssthresh  → skip slow start, go straight to congestion avoidance
```

## Example (Code)

### Example 1 — TCP Client with `getaddrinfo()` (IPv4 + IPv6 agnostic)

```c
/* tcp_getaddrinfo_client.c
 * Compile: gcc tcp_getaddrinfo_client.c -o tcp_client
 * Run:     ./tcp_client google.com 80
 *          ./tcp_client 127.0.0.1 8080
 *          ./tcp_client ::1 8080
 */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netdb.h>
#include <arpa/inet.h>

static void print_addr(const struct addrinfo *rp) {
    char host[NI_MAXHOST], port[NI_MAXSERV];
    if (getnameinfo(rp->ai_addr, rp->ai_addrlen,
                    host, sizeof(host), port, sizeof(port),
                    NI_NUMERICHOST | NI_NUMERICSERV) == 0)
        printf("  Trying [%s]:%s ...\n", host, port);
}

int main(int argc, char *argv[]) {
    if (argc != 3) {
        fprintf(stderr, "Usage: %s <host> <port>\n", argv[0]);
        exit(1);
    }

    struct addrinfo hints;
    memset(&hints, 0, sizeof(hints));
    hints.ai_family   = AF_UNSPEC;    /* accept IPv4 or IPv6 */
    hints.ai_socktype = SOCK_STREAM;  /* TCP */

    struct addrinfo *res;
    int err = getaddrinfo(argv[1], argv[2], &hints, &res);
    if (err != 0) {
        /* getaddrinfo has its own error code space — use gai_strerror */
        fprintf(stderr, "getaddrinfo: %s\n", gai_strerror(err));
        exit(1);
    }

    /* Iterate linked list — try each address until one connects */
    int sockfd = -1;
    for (struct addrinfo *rp = res; rp != NULL; rp = rp->ai_next) {
        print_addr(rp);
        sockfd = socket(rp->ai_family, rp->ai_socktype, rp->ai_protocol);
        if (sockfd == -1) continue;

        if (connect(sockfd, rp->ai_addr, rp->ai_addrlen) == 0)
            break;          /* success */

        close(sockfd);
        sockfd = -1;
    }
    freeaddrinfo(res);      /* always free, even if connection failed */

    if (sockfd == -1) {
        fprintf(stderr, "Could not connect to %s:%s\n", argv[1], argv[2]);
        exit(1);
    }
    printf("Connected.\n");

    /* Minimal HTTP/1.0 GET to demonstrate the connection */
    char req[256];
    snprintf(req, sizeof(req), "GET / HTTP/1.0\r\nHost: %s\r\n\r\n", argv[1]);
    write(sockfd, req, strlen(req));

    char buf[512] = {0};
    ssize_t n = read(sockfd, buf, sizeof(buf) - 1);
    if (n > 0) printf("First %zd bytes:\n%.200s\n", n, buf);

    close(sockfd);
    return 0;
}
```

### Example 2 — TCP Server with `getaddrinfo()` (dual-stack)

```c
/* tcp_getaddrinfo_server.c
 * Compile: gcc tcp_getaddrinfo_server.c -o tcp_server
 * Run:     ./tcp_server 8080
 */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netdb.h>
#include <arpa/inet.h>

int main(int argc, char *argv[]) {
    if (argc != 2) { fprintf(stderr, "Usage: %s <port>\n", argv[0]); exit(1); }

    struct addrinfo hints;
    memset(&hints, 0, sizeof(hints));
    hints.ai_family   = AF_UNSPEC;   /* AF_INET6 to force IPv6-only */
    hints.ai_socktype = SOCK_STREAM;
    hints.ai_flags    = AI_PASSIVE;  /* NULL host + AI_PASSIVE = INADDR_ANY */

    struct addrinfo *res;
    int err = getaddrinfo(NULL, argv[1], &hints, &res);
    if (err != 0) { fprintf(stderr, "getaddrinfo: %s\n", gai_strerror(err)); exit(1); }

    int listenfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);
    if (listenfd == -1) { perror("socket"); exit(1); }

    int opt = 1;
    setsockopt(listenfd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    if (bind(listenfd, res->ai_addr, res->ai_addrlen) == -1) {
        perror("bind"); exit(1);
    }
    freeaddrinfo(res);

    if (listen(listenfd, 10) == -1) { perror("listen"); exit(1); }
    printf("Listening on port %s\n", argv[1]);

    while (1) {
        /* sockaddr_storage is large enough for IPv4 or IPv6 client address */
        struct sockaddr_storage clientaddr;
        socklen_t addrlen = sizeof(clientaddr);
        int connfd = accept(listenfd, (struct sockaddr *)&clientaddr, &addrlen);
        if (connfd == -1) { perror("accept"); continue; }

        /* getnameinfo: binary address → human-readable string */
        char host[NI_MAXHOST], port[NI_MAXSERV];
        getnameinfo((struct sockaddr *)&clientaddr, addrlen,
                    host, sizeof(host), port, sizeof(port),
                    NI_NUMERICHOST | NI_NUMERICSERV);
        printf("Client connected: [%s]:%s\n", host, port);

        char buf[1024];
        ssize_t n;
        while ((n = read(connfd, buf, sizeof(buf))) > 0)
            write(connfd, buf, n);   /* echo */

        close(connfd);
    }

    close(listenfd);
    return 0;
}
```

### Example 3 — `inet_pton` / `inet_ntop` + Byte Order Demo

```c
/* inet_demo.c — Address conversion and byte order visualization
 * Compile: gcc inet_demo.c -o inet_demo
 * Run:     ./inet_demo
 */
#include <stdio.h>
#include <string.h>
#include <stdint.h>
#include <arpa/inet.h>
#include <netinet/in.h>

static void dump_hex(const char *label, const void *p, size_t n) {
    const uint8_t *b = p;
    printf("  %-34s: ", label);
    for (size_t i = 0; i < n; i++) printf("%02X ", b[i]);
    printf("\n");
}

int main(void) {
    printf("=== Byte Order (port 8080 = 0x1F90) ===\n");
    uint16_t host_port = 8080;
    uint16_t net_port  = htons(host_port);
    dump_hex("host byte order", &host_port, 2);   /* x86: 90 1F */
    dump_hex("network byte order", &net_port, 2); /* all: 1F 90 */

    printf("\n=== IPv4 Conversion ===\n");
    struct in_addr v4;
    inet_pton(AF_INET, "192.168.1.100", &v4);
    dump_hex("192.168.1.100 binary (net order)", &v4, 4); /* C0 A8 01 64 */

    char str4[INET_ADDRSTRLEN];
    inet_ntop(AF_INET, &v4, str4, sizeof(str4));
    printf("  inet_ntop result              : %s\n", str4);

    printf("\n=== IPv6 Conversion ===\n");
    struct in6_addr v6;
    inet_pton(AF_INET6, "2001:db8::1", &v6);
    dump_hex("2001:db8::1 binary", &v6, 16);

    char str6[INET6_ADDRSTRLEN];
    inet_ntop(AF_INET6, &v6, str6, sizeof(str6));
    printf("  inet_ntop result              : %s\n", str6);

    printf("\n=== Filling sockaddr_in Correctly ===\n");
    struct sockaddr_in sa;
    memset(&sa, 0, sizeof(sa));
    sa.sin_family = AF_INET;
    sa.sin_port   = htons(8080);                   /* MUST convert to network order */
    inet_pton(AF_INET, "10.0.0.1", &sa.sin_addr);  /* inet_pton already network order */
    printf("  sin_family  : %d (AF_INET = %d)\n", sa.sin_family, AF_INET);
    printf("  sin_port    : 0x%04X  (htons(8080))\n", sa.sin_port);
    dump_hex("  sin_addr   ", &sa.sin_addr, 4);

    printf("\n=== Special Addresses ===\n");
    printf("  INADDR_ANY       (htonl): 0x%08X → bind all interfaces\n",
           htonl(INADDR_ANY));
    printf("  INADDR_LOOPBACK  (htonl): 0x%08X → 127.0.0.1\n",
           htonl(INADDR_LOOPBACK));

    return 0;
}
```

## Debugging

### Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `ECONNREFUSED` | No server listening on that port | Start server, verify port |
| `ETIMEDOUT` | Host unreachable or firewalled | Check routing, firewall rules |
| `ENETUNREACH` | No route to host | Check network interface is up |
| Wrong port/address value | Forgot `htons()`/`htonl()` | Always convert before writing into `sockaddr` |
| `getaddrinfo` returns `EAI_NONAME` | Hostname not found in DNS | Check DNS, try numeric IP |
| `getaddrinfo` returns `EAI_SERVICE` | Service name unknown | Check `/etc/services` or use numeric port |
| `EMSGSIZE` on UDP | Datagram too large → IP fragmentation triggered | Keep UDP payload ≤ 512 bytes |

### Inspection Commands

```bash
# Show all TCP connections with state
ss -tanp

# Check what a hostname resolves to (uses same resolution as getaddrinfo)
getent hosts example.com

# DNS lookup directly
dig example.com A
dig example.com AAAA

# Check TCP send/receive buffer sizes and socket options
ss -tnop | grep <port>

# View TCP retransmission stats
netstat -s | grep -i retransmit

# Trace DNS + connect calls
strace -e trace=socket,connect,sendto,recvfrom,close ./tcp_client example.com 80

# Check TCP congestion window of active connection
ss -tnoi dst <ip>
# Output: cwnd:10 ssthresh:65535 rtt:0.5/0.25 ...

# View ephemeral port range
cat /proc/sys/net/ipv4/ip_local_port_range
```

## Real-world Usage

| Application | Protocol | Key sockaddr usage |
|-------------|----------|--------------------|
| **Web browsers** | `AF_INET/AF_INET6 SOCK_STREAM` (HTTP/HTTPS) | `getaddrinfo()` with hostname + "443" |
| **nginx / Apache** | `AF_INET6 SOCK_STREAM` (dual-stack) | `AI_PASSIVE` + `bind` to `::` |
| **DNS resolver (bind9)** | `AF_INET SOCK_DGRAM` port 53 | `recvfrom()` with `sockaddr_storage` |
| **Redis** | `AF_INET SOCK_STREAM` port 6379 | Keep-alive + `TCP_NODELAY` |
| **NTP** | `AF_INET SOCK_DGRAM` port 123 | Fixed 48-byte UDP datagrams |
| **SSH** | `AF_INET SOCK_STREAM` port 22 | `getaddrinfo()` + `AI_ADDRCONFIG` |

**Connection pooling and keep-alive**: because each new TCP connection starts slow-start from
a small cwnd, long-running connections reuse an already-warmed-up connection and avoid
handshake overhead. HTTP/1.1 `Connection: keep-alive`, HTTP/2 multiplexing, and database
connection pools all exploit this.

**`AI_ADDRCONFIG` hint**: tells `getaddrinfo()` to only return IPv4 results if the host has
an IPv4 address configured, and IPv6 results only if IPv6 is configured. Prevents useless
connection attempts to address types that have no local interface.

## Key Takeaways

| Concept | Core point |
|---------|-----------|
| TCP/IP is layered | Each layer encapsulates the layer above; lower layers treat upper data as opaque |
| IP | Connectionless + unreliable + best-effort: routing only, no delivery guarantee |
| TCP | Reliable byte-stream: sequencing, ACK/retransmit, flow control, congestion control |
| UDP | Port numbers + checksum on top of IP — unreliable, fast, no connection |
| 4-tuple | `(src_ip, src_port, dst_ip, dst_port)` uniquely identifies one TCP connection |
| Port ranges | 0–1023: privileged; 32768–60999: Linux ephemeral defaults |
| Byte order | Always `htons`/`htonl` writing to `sockaddr`; `ntohs`/`ntohl` reading back |
| `inet_pton/ntop` | Thread-safe, handles IPv4 + IPv6; replaces obsolete `inet_aton/ntoa` |
| `getaddrinfo()` | Modern lookup: thread-safe, IPv4+IPv6, DNS+/etc/hosts — returns linked list |
| Iterate list | Never pick only `res[0]` — loop and try each address until `connect()` succeeds |
| `sockaddr_storage` | Generic buffer for `accept()` / `recvfrom()` when IP version is unknown |
| UDP size | Keep payload ≤ 512 bytes to avoid IP fragmentation and silent data loss |
| Nagle / `TCP_NODELAY` | Disable Nagle for interactive protocols where latency > throughput |
| Slow start | New TCP connections start slow — connection reuse (keep-alive) avoids this cost |
